## מגבלות קריטיות

### 1. Selection Bias — רק הצבעות שעברו
`View_vote_rslts_hdr_Approved` מכיל **רק** הצבעות שאושרו.  
הצעות שהוכשלו על-ידי הקואליציה נעדרות לחלוטין.  
→ לא ניתן לנתח עמדות אופוזיציה בצורה שלמה.

### 2. כנסת 25 = אפס נתונים
האנדפוינט לא מכיל שום הצבעה מכנסת ה-25 — הכנסת הנוכחית.  
→ המפלגות הנוכחיות (הליכוד הנוכחי, הציונות הדתית, המחנה הממלכתי, יש עתיד וכו') **אינן** ניתנות לניתוח מהמאגר.

### 3. מיפוי למפלגות נוכחיות — בלתי ישיר
שמות הסיעות היסטוריים לא תואמים מפלגות נוכחיות:
- "כחול לבן" → לא קיימת
- "ימינה" → פוזלה
- "הבית היהודי" → בחלקה הפכה ל"הציונות הדתית"

### 4. נתוני פר-חכ חלקיים
K23: מצופה ~200,000 שורות, בפועל 108,346 (~54%).

## ממצא חיובי: רקורד חכ"ים עוקב אחריהם

הטבלה `vote_rslts_kmmbr_shadow` שומרת את **שם הסיעה בזמן ההצבעה** — לא הסיעה הנוכחית.  
כלומר: כשחכ עובר מפלגה, הצבעותיו הישנות נשארות תחת הסיעה הישנה, והחדשות תחת החדשה.

**דוגמאות שנבדקו**:

| חכ | מסלול | אמינות הנתון |
|----|-------|-------------|
| נפתלי בנט | K19–K20 הבית היהודי → K23–K24 ימינה | ✅ נכון — K23 FOR גבוה (ראש ממשלה), K24 AGAINST (אופוזיציה) |
| גדעון סער | K16–K22 ליכוד → K24 תקווה חדשה | ✅ מפריד נכון לפי סיעה בזמן |
| אביגדור ליברמן | תמיד ישראל ביתנו (חוץ מ-K19 ליכוד ביתנו) | ✅ הAPI מפריד את המיזוג הזמני נכון |

→ **כלי שמנתח חכ ספציפי לאורך ריצה יכול להשתמש בנתון זה בצורה אמינה.**
